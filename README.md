# adamant-spec

The canonical specification of Adamant Protocol.

This repository contains the formal whitepaper, design proposals, and protocol documentation. It is the source of truth for what Adamant is. Reference and alternative implementations conform to this specification.

## Status

Whitepaper v0.1 — complete.
Reference implementation — in progress (see [adamant](https://github.com/adamant-protocol/adamant)).

## Contents

- `whitepaper/` — full protocol specification, 13 sections
  - `00-abstract.md` through `12-conclusion.md` — individual sections
  - `adamant-whitepaper-complete.md` — single-file consolidated version
- `proposals/` — design proposals and amendments
- `rfcs/` — request-for-comment documents (forthcoming)

## Versioning

The whitepaper follows semantic versioning at the document level. v0.1 is the current specification. Material protocol changes increment the version; clarifications and corrections increment the minor or patch number.

## Changes

The protocol's core design is fixed at the specification level. Changes after launch require socially-coordinated hard forks per Section 11 of the whitepaper.

Pre-launch, changes happen via discussion in this repository. Open an issue or pull request for proposed amendments.
